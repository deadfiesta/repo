Download UI Screens

App write-up

Skootar is an express intra-city delivery service catered for both personal and corporate customers. The app allow parcel to be delivered quickly by finding the best match from available courier based on the customer's order.

UI Screens

  • 1.0 Welcome Popup

    I've added the get started button so user can directly start a new order rather than closing out the message to get to the Skootar home to create an order. There is a back caret button to return back to AirPay.

  • 2.0 Skootar Home (Empty)

    No changes or additional element.

  • 3.1 Skootar Home (Order List)

    I've grouped the order list into 3 tabs: Active, completed and all orders so user can quickly focus on active orders or review past completed orders only.

  • 3.2 Skootar Home (Order List) [Continue]

    I've also re-arranged the order of the info in terms of their visual hierarchy. (i.e. user can quickly see the status of the order first, followed by the recipient the order is sent for and also prices for that order)

  • 4.1 Create New Order

    I've rearranged the order of some of the info. (i.e. user selects their type of parcel first and everything related to picking up is grouped in one section, sender's name, pick up address and pick up time. Likewise for the info on the delivery.

  • 4.2 Create New Order [Continue]

    Additional create new order screen to show the ui when the user scroll to the end of the page before submitting the order.

  • 5.1 Order Submitted

    No changes to the default hierarchy or order of info displayed.

  • 5.2 Courier Found

    No changes to the default hierarchy or order of info displayed.

  • 5.3 Item Collected

    No changes to the default hierarchy or order of info displayed.

  • 5.4 Item Collected
